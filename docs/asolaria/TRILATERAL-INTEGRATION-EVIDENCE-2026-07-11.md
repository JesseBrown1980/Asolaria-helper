# Trilateral evidence for the Asolaria helper integration — 2026-07-11

Canonical doctrine:
[`HYPER-BECHS--the-third-set/TRILATERAL-REALITY-EVIDENCE-DOCTRINE-2026-07-11.md`](https://github.com/JesseBrown1980/HYPER-BECHS--the-third-set/blob/main/TRILATERAL-REALITY-EVIDENCE-DOCTRINE-2026-07-11.md)

This doctrine applies to the Asolaria/Hermes integration layer in this fork. It does not rewrite the
upstream helper project's identity, ownership, or independent feature claims.

## Integration states

```text
upstream source present
Asolaria adapter/config present
credential-free tests pass
local route/helper starts
fabric handoff emits evidence
operator-authorized task completes
```

Each state is real in its own scope. Upstream source is not a live Asolaria route. A staged adapter is
not nonexistent. A successful local helper is not proof of every remote provider or fabric path.

## Trilateral verification

- Acer records the integration source, local route, and any owning-seat runtime receipt.
- Liris attacks handoff semantics, safety boundaries, and byte/protocol parity.
- A third seat runs public integration tests with its own clean environment.
- CI verifies credential-free behavior against the immutable commit.

No third seat receives private task history, API keys, operator memory, or deeper-tier fabric data.

## No-deflate / no-inflate

Reject both:

```text
"forked upstream base, therefore Asolaria integration is unreal" -> deflation
"adapter exists, therefore live fabric handoff is proven"        -> inflation
"no private corpus in repo, therefore helper has no body"        -> deflation
"one local task passed, therefore all providers/routes work"     -> inflation
```

## Authority

The helper may prepare, route, or execute within its configured contract. It does not acquire apex,
cosign, mint, device-write, or autonomous scale authority from a third-seat or CI pass.

## Merge rule

Merge additive adapters, tests, protocol fixes, privacy hardening, and scoped receipts when clean.
Hold secrets, private task bodies, broad autonomous execution, authority changes, provider-account
claims, or live-fabric promotion without an owning-seat receipt.
